# Triage Label Taxonomy

Use this label set to keep issue and PR triage consistent.

## Type labels

- `type:bug` - Defect or regression
- `type:feature` - New capability
- `type:docs` - Documentation-only work
- `type:refactor` - Internal code structure changes
- `type:security` - Security-related work
- `type:test` - Test-only changes
- `type:chore` - Maintenance and tooling

## Priority labels

- `priority:P0` - Critical outage, auth bypass, data-loss risk
- `priority:P1` - High-impact user-facing issue
- `priority:P2` - Standard defect or enhancement
- `priority:P3` - Low urgency or backlog item

## Area labels

- `area:core`
- `area:adapters`
- `area:client`
- `area:dashboard`
- `area:cli`
- `area:docs`
- `area:plugins`
- `area:release`

## State labels

- `state:needs-repro` - Missing reproducible steps
- `state:needs-design` - Requires design decision
- `state:blocked` - Waiting on external dependency
- `state:ready` - Ready for implementation
- `state:in-progress` - Actively being worked
- `state:needs-review` - Waiting on maintainer review

## Good first contribution labels

- `good-first-issue` - Beginner friendly task
- `help-wanted` - Community contribution requested

## Process guidance

1. Apply one `type:*` label.
2. Apply one `priority:*` label.
3. Apply one or more `area:*` labels.
4. Add state labels as work progresses.
5. Add `good-first-issue` only when scope is small and well documented.
