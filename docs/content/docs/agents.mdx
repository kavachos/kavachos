## What is an agent identity

An `AgentIdentity` is the primary entity in KavachOS. It represents a single AI agent — a process that acts on behalf of a human user. Every agent has:

- A stable `id` (prefixed `agt_`) that never changes
- A bearer token (`kv_...`) that can be rotated
- A set of permissions that define what it can do
- A status: `active`, `revoked`, or `expired`
- An owner: the `userId` of the human responsible for it

Agents are not users. They do not have passwords, sessions, or OAuth flows. They have tokens and permissions.

## Creating agents

```typescript
const agent = await kavach.agent.create({
  ownerId: 'user-123',   // ID from your auth provider
  name: 'github-reader',
  type: 'autonomous',
  permissions: [
    { resource: 'mcp:github:*', actions: ['read'] },
  ],
  expiresAt: new Date(Date.now() + 7 * 24 * 3_600_000), // optional expiry
  metadata: { purpose: 'nightly PR review' },            // arbitrary key/value
});
```

The returned object contains the plaintext token exactly once. KavachOS stores only the SHA-256 hash, so the token cannot be recovered later — only rotated.

### Agent types

**`autonomous`** — Acts independently without requiring human approval on each call (unless a permission constraint mandates it). The standard type for background agents, cron jobs, and AI assistants.

**`delegated`** — Receives permissions from another agent via a delegation chain rather than having them declared at creation time. Use this for ephemeral sub-agents spun up to complete a specific task.

**`service`** — Long-lived identity for a piece of infrastructure, such as an MCP server or an internal microservice that needs to call other services on behalf of users.

## Agent tokens

Tokens use the `kv_` prefix followed by 32 cryptographically random bytes encoded as hex. The token is hashed with SHA-256 before storage, so even a full database dump cannot recover active tokens.

```
kv_a3f8c2e1d4b5...   (64 hex chars after the prefix)
```

To authenticate as an agent, pass the token as a Bearer credential:

```
Authorization: Bearer kv_a3f8c2e1d4b5...
```

## Token rotation

Rotation issues a new token and immediately invalidates the old one. The operation is atomic — there is no window where both tokens are valid.

```typescript
const rotated = await kavach.agent.rotate(agentId);
// rotated.token is the new plaintext token
// The old token is rejected immediately
```

Rotate tokens periodically, or whenever you suspect a token has been exposed.

## Token-based authorization

When the caller only has a raw bearer token (for example, in an HTTP middleware), use `authorizeByToken` instead of `authorize`:

```typescript
// In an HTTP handler
const token = request.headers.get('Authorization')?.replace('Bearer ', '');
if (!token) return new Response('Unauthorized', { status: 401 });

const result = await kavach.authorizeByToken(token, {
  action: 'read',
  resource: 'mcp:github:repos',
});

if (!result.allowed) {
  return new Response(result.reason ?? 'Forbidden', { status: 403 });
}
```

This performs a single database lookup (hash comparison), then evaluates permissions in memory. No JWTs, no network calls.

## Listing and filtering agents

```typescript
// All agents owned by a user
const agents = await kavach.agent.list({ userId: 'user-123' });

// Only active autonomous agents
const active = await kavach.agent.list({
  userId: 'user-123',
  status: 'active',
  type: 'autonomous',
});
```

## Updating an agent

```typescript
const updated = await kavach.agent.update(agentId, {
  name: 'github-reader-v2',
  permissions: [
    { resource: 'mcp:github:*', actions: ['read', 'comment'] },
  ],
});
```

Permission updates take effect immediately. Ongoing requests that have already passed authorization are not affected.

## Revoking an agent

```typescript
await kavach.agent.revoke(agentId);
// All future authorize() or authorizeByToken() calls for this agent return allowed: false
// The agent's token is also invalidated
```

Revocation is permanent. To re-enable an agent, create a new one.

## Max agents per user

The default limit is 10 active agents per user. Change it at initialization:

```typescript
const kavach = createKavach({
  database: { provider: 'sqlite', url: 'kavach.db' },
  agents: {
    maxPerUser: 50,  // raise the limit
  },
});
```

Attempts to create agents beyond the limit return an error with code `AGENT_LIMIT_EXCEEDED`.
