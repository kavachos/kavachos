## Session management

KavachOS provides cookie-based session management for human users alongside bearer tokens for agents.

### Cookie sessions

```typescript
import { createCookieSessionManager } from 'kavachos';

const sessions = createCookieSessionManager({
  sessionName: 'kavach_session',
  maxAge: 7 * 24 * 60 * 60, // 7 days
  autoRefresh: true,
  cookie: {
    httpOnly: true,
    secure: true,
    sameSite: 'lax',
  },
}, kavach.db);

// Create session after login
const { session, setCookieHeader } = await sessions.createSession(userId);

// Validate on each request
const { session: valid } = await sessions.validateSession(req.headers.cookie);

// Revoke
await sessions.revokeSession(sessionId);
```

### CSRF protection

KavachOS uses the double-submit cookie pattern with timing-safe comparison:

```typescript
import { generateCsrfToken, validateCsrfToken, validateOrigin } from 'kavachos';

// Generate token and set as cookie
const token = generateCsrfToken();

// Validate on mutation requests
const valid = validateCsrfToken(requestToken, cookieToken);

// Also validate origin header
const originOk = validateOrigin(request, ['https://myapp.com']);
```

### Human sessions vs agent tokens

| | Human sessions | Agent tokens |
|---|---|---|
| Format | Signed JWT in httpOnly cookie | `kv_...` bearer token |
| Lifetime | 7 days (configurable) | 24 hours (configurable) |
| Storage | Database row | SHA-256 hash in database |
| Revocation | Per-session or per-user bulk | Per-agent |
| CSRF | Required (cookie-based) | Not needed (header-based) |
