## How audit logging works

Every call to `kavach.authorize()` or `kavach.authorizeByToken()` is written to the audit log, regardless of outcome. The log is append-only — entries are never updated or deleted. Even denied calls, rate-limited calls, and calls from revoked agents are recorded.

Each entry captures:

```typescript
interface AuditEntry {
  id: string;                                         // aud_...
  agentId: string;
  userId: string;
  action: string;
  resource: string;
  parameters: Record<string, unknown>;               // arguments passed to the tool
  result: 'allowed' | 'denied' | 'rate_limited';
  durationMs: number;                                // time spent evaluating the decision
  tokensCost?: number;                               // optional token usage from the agent's LLM call
  timestamp: Date;
}
```

The `auditId` returned by `authorize()` links the authorization decision to its log entry:

```typescript
const result = await kavach.authorize(agent.id, {
  action: 'read',
  resource: 'mcp:github:repos',
});

console.log(result.auditId);  // "aud_3f8a..."
```

## Querying audit logs

```typescript
const logs = await kavach.audit.query({
  agentId: 'agt_...',
  userId: 'user-123',
  result: 'denied',
  since: new Date('2025-01-01'),
  until: new Date('2025-02-01'),
  actions: ['write', 'delete'],
  limit: 100,
  offset: 0,
});
```

All filter fields are optional and can be combined. Without any filters, the query returns all entries up to the `limit` (default: 1000).

### Filter reference

| Field | Type | Description |
|---|---|---|
| `agentId` | `string` | Filter to a specific agent |
| `userId` | `string` | Filter to all agents owned by a user |
| `since` | `Date` | Entries on or after this timestamp |
| `until` | `Date` | Entries before this timestamp |
| `actions` | `string[]` | Filter to specific actions |
| `result` | `'allowed' \| 'denied' \| 'rate_limited'` | Filter by outcome |
| `limit` | `number` | Max entries to return (default: 1000) |
| `offset` | `number` | Pagination offset |

## Exporting logs

```typescript
// JSON export
const json = await kavach.audit.export({ format: 'json' });

// CSV export — suitable for spreadsheets and compliance tools
const csv = await kavach.audit.export({ format: 'csv' });

// Export a time range
const q4 = await kavach.audit.export({
  format: 'csv',
  since: new Date('2024-10-01'),
  until: new Date('2025-01-01'),
});
```

The CSV export includes a header row with field names matching the `AuditEntry` interface. Each row represents one authorization decision.

## Compliance references

The audit trail is designed to satisfy these requirements out of the box:

**EU AI Act — Article 12 (Record keeping)**
Article 12 requires high-risk AI systems to automatically log events throughout their lifecycle, including the period of activity and data used. KavachOS records every authorization decision with agent identity, resource, action, parameters, outcome, and timestamp.

**NIST AI RMF — GOVERN 1.7 / MANAGE 4.2**
The NIST AI Risk Management Framework calls for documented accountability mechanisms and the ability to trace AI actions to specific identities. The audit trail provides both.

**SOC 2 — CC6.1 through CC7.2**
SOC 2 trust services criteria for logical access controls and system monitoring require evidence that access is granted only to authorized identities and that access events are logged. The `allowed`/`denied`/`rate_limited` result field and agent identity fields satisfy CC6.1–CC6.3, and the tamper-evident append-only log satisfies CC7.2.

**ISO 42001 — Annex A.8 (Documentation of AI system behaviour)**
ISO 42001 recommends documenting the behavior of AI systems in production. Exporting the audit log as JSON or CSV provides a machine-readable record suitable for auditor review.

## Using audit data

A few common patterns:

```typescript
// Monthly access report for a user
const entries = await kavach.audit.query({
  userId: 'user-123',
  since: new Date('2025-03-01'),
  until: new Date('2025-04-01'),
});

// Find agents that have been denied more than once today
const denied = await kavach.audit.query({
  result: 'denied',
  since: new Date(new Date().setHours(0, 0, 0, 0)),
});

const agentDenyCounts = denied.reduce<Record<string, number>>((acc, entry) => {
  acc[entry.agentId] = (acc[entry.agentId] ?? 0) + 1;
  return acc;
}, {});

// Identify high-frequency agents (potential runaway loops)
const highActivity = await kavach.audit.query({
  agentId: 'agt_...',
  since: new Date(Date.now() - 3_600_000),  // last hour
});
console.log(`${highActivity.length} calls in the last hour`);
```
